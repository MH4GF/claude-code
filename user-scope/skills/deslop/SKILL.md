# Deslop

main ブランチとの diff を読み、AI 生成コードに紛れがちなパターンを検出して指摘する。**ファイル編集は行わない**。修正判断は呼び出し元に委ねる。

## 検出対象

- 周辺コードのスタイルと一致しない不要なコメント
- 信頼できる内部パスでの過剰な try/catch・防御チェック
- 型エラー回避だけが目的の `any` キャスト
- 早期リターンで平坦化できる深いネスト
- ファイル全体の慣習と乖離した書き方

## 手順

1. ベースブランチを判定する（既定: `main`、無ければ `master`）
2. `git diff <base>...HEAD` で変更内容を取得する
3. 検出対象に該当する箇所をファイルパス・行番号付きでリスト化する
4. 各指摘に「なぜノイズか」と「推奨書き換え」を 1〜2 行で添える

## 出力フォーマット

```
## Deslop 指摘
- <path>:<line> — <カテゴリ> — <推奨書き換え>
...

## サマリ
<1〜3 文。検出ゼロなら「該当なし」とだけ書く>
```

## ガードレール

- 振る舞いが変わる可能性のある指摘は明示的にフラグを立てる
- 大規模な書き換え提案は出さない。最小・局所修正のみ
